# Specification: Platform Backend (FastAPI)

**Track ID:** platform-backend_20260408
**Type:** Feature
**Created:** 2026-04-08
**Status:** Draft

## Summary

Backend FastAPI da plataforma conversacional. Gerencia auth multi-tenant (modelo IAM), contexto do pipeline, orquestracao LLM com tools, GitHub PRs, web chat SSE, integracao com Omni, e persistencia de conversas. Cada empresa fornece suas proprias credenciais (Anthropic, Discord, Telegram, WhatsApp, GitHub, Databricks) via pagina de settings.

## User Story

Como admin de uma empresa, quero configurar minhas credenciais (Anthropic, canais, GitHub, Databricks) e criar contas para minha equipe, para que todos possam conversar com o agente sobre nossos pipelines.

## Acceptance Criteria

### Auth & Multi-Tenant (modelo IAM)
- [ ] Conta root da empresa: cria empresa, configura settings, gerencia usuarios
- [ ] Usuarios criados pelo root/admin (nao self-register)
- [ ] RBAC: root > admin > editor > viewer
- [ ] JWT + refresh token (httpOnly cookies), OAuth2 opcional
- [ ] Isolamento total: toda query filtra por company_id via middleware
- [ ] Credenciais da empresa criptografadas (Fernet, chave separada do JWT)

### Settings da Empresa (root/admin only)
- [ ] Anthropic API Key — a empresa paga pelo seu uso de LLM
- [ ] Discord Bot Token — opcional, para canal Discord
- [ ] Telegram Bot Token — opcional, para canal Telegram
- [ ] WhatsApp QR Code — opcional, pairing via Omni WebSocket na pagina de settings
- [ ] GitHub Repo + Token — para PRs automaticos do agente
- [ ] Databricks Host + Token — conexao com o pipeline da empresa
- [ ] Todas as credenciais criptografadas em repouso (Fernet)
- [ ] Validacao de credenciais no momento do save (test connection)

### Chat & LLM
- [ ] POST /chat/message retorna SSE stream
- [ ] Context Engine: 3 niveis (resumo 800tk, detalhes 8k, completo 35k)
- [ ] Claude API com tool use (query_table, get_status, create_pr, trigger_run, send_email)
- [ ] Acoes perigosas requerem confirmacao inline
- [ ] Modelo selecionado pela empresa: Sonnet (default) ou Opus

### Webhook Omni
- [ ] POST /webhooks/omni recebe mensagens normalizadas
- [ ] Identifica usuario por channel_identity (phone, discord_id, telegram_id)
- [ ] Slash commands: /resume, /pipelines, /status, /threads, /new, /help
- [ ] Resposta enviada de volta via Omni API

### Omni Management (automatizado pelo backend)
- [ ] Na configuracao de canal (settings), backend chama Omni API para:
  - Criar instancia (whatsapp-baileys, discord, telegram)
  - Passar token do canal
  - Configurar webhook provider apontando para nosso backend
  - Configurar agent routes
- [ ] QR Code do WhatsApp: Omni gera, backend proxeia via WebSocket para o frontend

### Database (PostgreSQL)
- [ ] companies: id, name, slug, settings_encrypted, created_at
- [ ] users: id, company_id, email, password_hash, name, role, is_active
- [ ] pipelines: id, company_id, name, databricks_job_id, github_repo
- [ ] threads: id, pipeline_id, user_id, title, is_active
- [ ] messages: id, thread_id, role, content, actions, channel, model, token_count
- [ ] active_sessions: user_id, channel, active_thread_id, active_pipeline_id
- [ ] channel_identities: user_id, channel, channel_user_id, verified
- [ ] company_credentials: company_id, credential_type, encrypted_value
- [ ] pipeline_context_cache: pipeline_id, context_type, content, token_estimate

### Infra
- [ ] Redis: cache L1 (contexto 60s), sessoes, rate limiting
- [ ] Metricas: /metrics (Prometheus)
- [ ] Testes: pytest-asyncio (auth, chat, context engine, tools, webhook)

## Technical Notes

- Padroes idlehub: AuthContext, Service layer async, domain exceptions, Pydantic Settings cascade, lifespan hooks, Fernet encryption separada do JWT
- Credenciais da empresa nunca em texto claro no DB — sempre Fernet encrypted
- Validacao de credenciais: ao salvar Anthropic key, faz test call; ao salvar Databricks token, faz test connection
- QR Code WhatsApp: Omni expoe WebSocket para QR, backend proxeia para frontend via SSE ou WS
- Token budget LLM: 80k max, Anthropic key da empresa usada (nao a nossa)
