# Specification: Observer — Multi-File Fixes

**Track ID:** observer-multifile_20260409
**Type:** Feature
**Created:** 2026-04-09
**Status:** Draft
**Priority:** 7

## Summary

Permitir que o LLM proponha correcoes em multiplos arquivos em um unico PR. Hoje o Observer so suporta fix em 1 arquivo. Bugs que cruzam modulos (ex: schema change em contracts.py + ajuste em ingest.py) precisam de multi-file.

## Context

O `DiagnosisResult` atual tem `fixed_code: str | None` e `file_to_fix: str | None` — singular. O `GitProvider.create_fix_pr()` cria commit com 1 arquivo. Para bugs cross-module, precisamos suportar lista de (file_path, code).

## User Story

Como engenheiro de dados, quero que o Observer possa corrigir bugs que afetam multiplos arquivos em um unico PR, para que a correcao seja atomica e completa.

## Acceptance Criteria

- [ ] `DiagnosisResult` suporta lista de arquivos/codigos (retrocompativel)
- [ ] LLM prompt solicita multi-file quando contexto indica necessidade
- [ ] `GitProvider.create_fix_pr()` cria commit com N arquivos
- [ ] PR mostra diff de todos os arquivos modificados
- [ ] Fix de 1 arquivo continua funcionando (retrocompativel)

## Dependencies

Nenhuma.

## Out of Scope

- Criacao de arquivos novos (apenas modificacao de existentes)
- Delecao de arquivos
- Resolucao de conflitos de merge

## Technical Notes

### Mudanca no DiagnosisResult

```python
@dataclass
class DiagnosisResult:
    # Campos existentes (retrocompativel)
    fixed_code: str | None = None
    file_to_fix: str | None = None

    # Novo campo para multi-file
    fixes: list[dict] | None = None  # [{"file_path": "...", "code": "..."}]
```

Se `fixes` preenchido, usar. Senao, fallback para `fixed_code`/`file_to_fix`.
