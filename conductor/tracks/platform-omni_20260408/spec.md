# Specification: Omni Integration + Multi-Channel

**Track ID:** platform-omni_20260408
**Type:** Feature
**Created:** 2026-04-08
**Status:** Draft

## Summary

Configuracao e integracao do Omni como gateway de canais externos (WhatsApp, Discord, Telegram). Inclui setup do servico, docker-compose, scripts de configuracao automatizada, e fluxo de pairing do WhatsApp QR Code via settings da empresa.

## User Story

Como admin da empresa, quero conectar canais de mensagem (WhatsApp, Discord, Telegram) na pagina de settings e ter minha equipe conversando com o agente por esses canais automaticamente.

## Acceptance Criteria

- [ ] Omni rodando como servico separado (docker-compose com PostgreSQL + NATS)
- [ ] Backend cria instancias no Omni automaticamente quando admin configura canal em settings
- [ ] Webhook provider configurado: Omni envia mensagens para POST /webhooks/omni do backend
- [ ] WhatsApp: pairing via QR Code na pagina de settings (WebSocket proxeado pelo backend)
- [ ] Discord: admin cola bot token → instancia criada → bot online
- [ ] Telegram: admin cola bot token → instancia criada → bot respondendo
- [ ] Agent routes configurados: todos os canais usam nosso webhook provider
- [ ] Mensagens chegam no backend normalizadas (formato Omni padrao)
- [ ] Respostas enviadas de volta pelo Omni ao canal correto
- [ ] Health check: backend monitora se Omni esta saudavel
- [ ] Desconectar canal: admin pode desativar canal em settings

## Dependencies

- Backend FastAPI rodando (Phase 2+ do Track platform-backend)
- Docker disponivel para rodar Omni + PostgreSQL + NATS

## Out of Scope

- Codigo-fonte do Omni (pacote externo, instalado via bun)
- WhatsApp Cloud API (oficial da Meta) — usamos Baileys (unofficial, sem aprovacao necessaria)
- Slack (futuro)

## Technical Notes

- Omni instalado via: bun add -g @automagik/omni ou docker
- Credenciais dos canais ficam no backend (criptografadas), enviadas ao Omni via API
- Omni usa seu proprio PostgreSQL (separado do backend) para estado de conexao
- NATS JetStream para eventos internos do Omni
- Para WhatsApp: Omni armazena auth state no PostgreSQL dele, reconecta automaticamente
- docker-compose.yml em platform/omni/ com: omni-api, omni-postgres, omni-nats
