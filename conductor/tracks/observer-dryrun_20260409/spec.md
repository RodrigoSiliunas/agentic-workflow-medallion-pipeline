## Summary

Widget `dry_run=true` que faz o Observer diagnosticar o erro e logar o resultado, mas NAO criar PR no GitHub. Util para validar setup, testar novos providers, ou ambientes de staging.

## Context

Ao configurar o Observer pela primeira vez ou testar novos LLM providers, o usuario pode querer ver o diagnostico sem criar PRs reais. Hoje a unica opcao e executar o Observer completo, o que cria branches e PRs no repositorio.

## User Story

Como engenheiro de dados, quero executar o Observer em modo dry-run para validar que o diagnostico funciona corretamente antes de habilitar a criacao automatica de PRs.

## Acceptance Criteria

- [ ] Widget `dry_run` disponivel no Observer (default: "false")
- [ ] Quando `dry_run=true`, Observer faz diagnostico completo via LLM
- [ ] Quando `dry_run=true`, Observer NAO cria branch/PR no GitHub
- [ ] Diagnostico e logado no output do notebook com detalhes completos
- [ ] Se observability estiver ativa, diagnostico dry-run e salvo na tabela com `status="dry_run"`

## Dependencies

Nenhuma (observability e opcional).

## Out of Scope

- UI para ativar/desativar dry-run (apenas widget)
- Dry-run parcial (ex: diagnosticar mas nao salvar)

## Technical Notes

- Widget: `dbutils.widgets.dropdown("dry_run", "false", ["true", "false"])`
- No notebook `collect_and_fix.py`, checar `dry_run` antes de chamar `git.create_fix_pr()`
- Logar diagnostico completo com `logger.info()` incluindo fixed_code
- Se tabela `observer.diagnostics` existir, salvar com `status="dry_run"`
