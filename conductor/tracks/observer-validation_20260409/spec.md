# Specification: Observer — Validacao Pre-PR

**Track ID:** observer-validation_20260409
**Type:** Feature
**Created:** 2026-04-09
**Status:** Draft
**Priority:** 6

## Summary

Rodar ruff lint + pytest no fix proposto pelo LLM ANTES de criar o PR. Se o fix nao passa nos checks, o Observer NAO cria PR e loga como "fix invalido". Aumenta confianca e evita PRs quebrados.

## Context

Hoje o Observer cria PRs sem validar se o codigo proposto pelo LLM e sintaticamente correto ou passa nos testes. Isso pode gerar PRs que falham no CI, desperdicando tempo de review humano. Validar antes de criar PR elimina esse problema.

## User Story

Como revisor de PRs, quero que PRs criados pelo Observer ja tenham passado por lint e testes basicos, para que eu nao perca tempo revisando codigo que nem compila.

## Acceptance Criteria

- [ ] Fix proposto e validado com ruff antes de criar PR
- [ ] Fix proposto e validado com pytest (testes relevantes) antes de criar PR
- [ ] Se validacao falha, PR NAO e criado
- [ ] Falha de validacao e logada com detalhes (qual check falhou, output)
- [ ] Se observability ativa, registro com `status="validation_failed"`

## Dependencies

Nenhuma (observability e opcional).

## Out of Scope

- Execucao do fix no Databricks (apenas validacao local/CI)
- Auto-fix do fix (LLM corrigir o proprio fix)
- Testes de integracao (apenas unit tests)

## Technical Notes

### Fluxo

```
LLM propoe fix → salva em arquivo temporario → ruff check → pytest → se OK → cria PR
                                                                    → se FAIL → loga e skip
```

### Implementacao

- Criar helper `validate_fix()` em `pipeline_lib/agent/observer/validator.py`
- Escrever `fixed_code` em arquivo temporario
- Rodar `subprocess.run(["ruff", "check", temp_file])` e capturar output
- Rodar `subprocess.run(["pytest", "pipeline/tests/", "-x", "--tb=short"])` e capturar output
- Retornar `(is_valid: bool, errors: list[str])`

### Desafio no Databricks

ruff e pytest podem nao estar instalados no cluster. Opcoes:
1. Instalar via `%pip install ruff pytest` no notebook Observer
2. Validar apenas sintaxe Python (`compile()`) + AST check como fallback
3. Delegar validacao para o CI (PR e criado como draft, CI valida)

Recomendacao: Opcao 1 (install no cluster) com fallback para Opcao 2.
