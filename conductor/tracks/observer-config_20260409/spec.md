# Specification: Observer — Configuracao como Codigo

**Track ID:** observer-config_20260409

## Summary

Arquivo de configuracao YAML/JSON no repositorio que define todos os parametros do Observer, ao inves de depender exclusivamente de widgets Databricks. Permite versionamento, code review, e reproducibilidade.

## Context

Hoje os parametros do Observer (provider, modelo, base_branch, etc.) sao definidos via widgets no notebook ou parametros do job. Isso dificulta versionamento, auditoria, e reproducao de configuracoes. Um arquivo de config no repo resolve esses problemas.

## User Story

Como engenheiro de dados, quero configurar o Observer via arquivo YAML no repositorio, para que mudancas de configuracao passem por code review e sejam versionadas junto com o codigo.

## Acceptance Criteria

- [ ] Arquivo `pipeline/observer_config.yaml` define todos os parametros
- [ ] Observer le config do YAML com fallback para widgets (retrocompativel)
- [ ] Campos: llm_provider, model, git_provider, base_branch, max_retries, dedup_window_hours, max_tokens_per_day, dry_run
- [ ] Validacao do YAML com Pydantic (erros claros se config invalida)
- [ ] Widgets continuam funcionando como override do YAML

## Dependencies

Nenhuma.

## Out of Scope

- UI para editar configuracao
- Hot-reload de config (requer re-execucao do Observer)
- Config por workflow (uma config global por ora)

## Technical Notes

### Formato do YAML

```yaml
# observer_config.yaml
observer:
  llm_provider: anthropic
  model: claude-opus-4-20250514
  max_tokens: 16000
  git_provider: github
  base_branch: dev
  max_retries: 3
  dedup_window_hours: 24
  max_tokens_per_day: 100000
  dry_run: false
  confidence_threshold: 0.5  # Abaixo disso, nao cria PR
```

### Hierarquia de Prioridade

1. Widgets Databricks (maior prioridade — override manual)
2. `observer_config.yaml` no repo
3. Defaults hardcoded no codigo (menor prioridade)

### Implementacao

- Pydantic model `ObserverConfig` em `pipeline_lib/agent/observer/config.py`
- Loader: tenta ler YAML do repo path, fallback para defaults
- Widgets sobrescrevem campos individuais se preenchidos
