# Specification: Observer — Deduplicacao de Diagnosticos

**Track ID:** observer-dedup_20260409
**Type:** Feature
**Created:** 2026-04-09
**Status:** Draft
**Priority:** 3

## Summary

Antes de chamar o LLM, verificar se o mesmo erro ja foi diagnosticado recentemente (hash do error message). Evita PRs duplicados e gasto desnecessario de tokens.

## Context

Se o pipeline falha repetidamente com o mesmo erro (ex: schema invalido), o Observer cria um PR novo a cada execucao. Isso gera PRs duplicados, gasta tokens, e polui o repositorio. Precisamos de um cache de diagnosticos.

## User Story

Como engenheiro de dados, quero que o Observer nao crie PRs duplicados para o mesmo erro, para evitar ruido no repositorio e gasto desnecessario de tokens.

## Acceptance Criteria

- [ ] Observer verifica cache antes de chamar LLM
- [ ] Mesmo erro dentro da janela de dedup (configurable, default 24h) nao gera novo diagnostico
- [ ] Cache usa hash SHA-256 do error message
- [ ] Se o PR anterior foi fechado sem merge, permite re-diagnostico
- [ ] Metricas de cache hit/miss registradas

## Dependencies

- **observer-observability_20260409** — usa a tabela `observer.diagnostics` como cache

## Out of Scope

- Dedup cross-workflow (apenas mesmo job_id)
- Machine learning para deteccao de erros similares (nao identicos)

## Technical Notes

- Hash: `hashlib.sha256(error_message.encode()).hexdigest()`
- Query: `SELECT * FROM observer.diagnostics WHERE error_hash = '{hash}' AND timestamp > current_timestamp() - INTERVAL '{window}' HOURS`
- Se encontrar diagnostico recente com PR aberto/mergeado → skip
- Se encontrar diagnostico recente com PR fechado (sem merge) → re-diagnosticar
- Janela default: 24 horas (configuravel via widget `dedup_window_hours`)
