## Summary

Implementar o agente autonomo (agent_pre + agent_post), o Databricks Workflow completo (7 tasks), sistema de notificacoes por email, auto-correcao com rollback Delta, e observabilidade (metricas, dashboard, alerts).

## User Story

Como avaliador, quero ver um pipeline que se auto-gerencia: detecta dados novos, processa automaticamente, se auto-corrige em caso de falha, e envia relatorios por email em todos os cenarios (sucesso, correcao, falha critica).

## Acceptance Criteria

- [ ] Notebook `agent_pre.py` (Task 0): verifica dados novos via S3 metadata, captura versoes Delta, seta task values
- [ ] Notebook `agent_post.py` (Task 5, run_if: ALL_DONE): verifica resultados, recovery, rollback, notificacoes
- [ ] Databricks Workflow configurado: Tasks 0→1→2a→2b/2c→3→4→5, max_concurrent_runs=1, cron diario
- [ ] Retries: 2x por task de dados, 0 para agent_post (guardrail)
- [ ] Auto-correcao: rollback Delta via `RESTORE TABLE ... TO VERSION AS OF`
- [ ] Guardrail: para apos 3 falhas consecutivas
- [ ] Email de sucesso com resumo de execucao
- [ ] Email de correcao com detalhes do que foi feito
- [ ] Email critico quando correcao falha ou 3+ falhas consecutivas
- [ ] Delta Tables: `pipeline.state`, `pipeline.metrics`, `pipeline.notifications`
- [ ] Dashboard Databricks SQL com 6 paineis
- [ ] Databricks SQL Alerts para falhas consecutivas e dias sem sucesso
